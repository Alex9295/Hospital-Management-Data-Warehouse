# 🏥 Hospital Operations Analytics (SQL Project)

## 📌 Project Overview

This project simulates a relational hospital management system and applies advanced SQL analytics to evaluate operational performance, financial metrics, and patient trends.

The objective is to demonstrate strong SQL proficiency, data cleaning skills, relational database understanding, and the ability to translate structured data into actionable business insights.

This project answers real-world questions such as:

- Which departments generate the highest revenue?
- What is the average patient length of stay?
- Which diagnoses have the highest readmission rates?
- How do admissions trend over time?
- Which physicians and departments are most utilized?

---

## 🛠️ Technologies Used

- **SQL (PostgreSQL)**
- Relational Database Design
- Data Cleaning & Validation
- Aggregations & KPI Reporting
- Window Functions
- Multi-table Joins

---

## 🗂️ Database Schema

The project includes a relational database consisting of:

- `Patients`
- `Doctors`
- `Departments`
- `Diagnoses`
- `Treatments`
- `PatientVisits`
- `Billing`
- `PaymentMethods`

Primary and foreign key relationships were implemented to simulate a realistic hospital data environment.
